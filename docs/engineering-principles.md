# Engineering Principles

## Explain WHY before HOW

Every implementation starts with the problem.

---

## Prefer simplicity

The simplest solution that satisfies the requirements wins.

---

## Composition over inheritance

Always prefer composition.

---

## Dependency inversion

Business rules must never depend on infrastructure.

---

## Reproducible environments

Every environment must be reproducible using code.

---

## Infrastructure is disposable

Infrastructure can always be recreated.

---

## Documentation is code

Documentation evolves together with the project.
